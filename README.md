# NaniTsukuru — キャンプ飯プランナー

少人数キャンプ向けの献立・買い物リスト作成アプリです。

朝昼晩の食事枠にレシピを割り当てると、必要な材料をカテゴリ別に集計して買い物リストを作成します。

## 特徴

- 食事枠（朝・昼・夜）へのレシピ割り当て（1枠に複数品まで追加可能）
- 1泊2日〜4泊5日の泊数に対応
- 人数変更で材料量を自動換算（目安）
- 材料をカテゴリ別に合算・買い物リスト生成
- 概算予算の自動集計
- タグ・名前・説明でレシピを検索
- アプリ内でレシピを追加（localStorage保存）
- 追加レシピのJSONエクスポート（`src/recipes.json`への正式採用用）
- 選択プランをURLで共有
- 印刷用レイアウト対応（食事プラン＋買い物リスト＋レシピ詳細）
- PWA対応（ホーム画面に追加可能）

## レシピについて

収録レシピはGW・群馬県嬬恋村のキャンプを想定して作成しています。標高が高く朝晩が冷える前提で、煮込み料理・温かいスープ・翌朝リメイクしやすいメニューを中心にしています。

基本レシピは2人分で作成しています。人数を変更すると材料量を目安として自動換算しますが、缶詰・調味料など購入単位のあるものは実際の量に合わせて調整してください。

## レシピを追加する

### アプリ内から追加

レシピ一覧の「レシピを追加」ボタンからフォームで入力できます。材料は「牛すじ 500g」「玉ねぎ 2個」形式で貼り付けると自動で解析します。追加したレシピはブラウザの localStorage に保存されます。

ブラウザをまたいで使いたい場合は「JSONエクスポート」でコピーし、`src/recipes.json` に追記してください。

### `src/recipes.json` に直接追加

材料は `itemId` と `unit` が同じものだけ合算します。玉ねぎ・バターなど複数レシピで共通する食材は、既存レシピと `itemId` を揃えると買い物リストがまとまります。

## ローカル実行

```bash
npm install
npm run dev
```

## GitHub Pages へのデプロイ

`.github/workflows/deploy.yml` が含まれています。

1. Settings → Pages → Source を `GitHub Actions` に変更
2. `main` ブランチへ push するとデプロイが実行されます
